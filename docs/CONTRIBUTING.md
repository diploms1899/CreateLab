# Contributing to CreateLab

## Engineering Standards

- Write production-quality code
- Prioritize maintainability
- Favor clean architecture over shortcuts
- Avoid duplicated logic
- Prefer composition over inheritance
- Design modules with low coupling and high cohesion
- Document important architectural decisions

## Development Process

1. Check existing architecture docs
2. Make changes in small, focused commits
3. Write tests for new functionality
4. Run existing test suite before PR
5. Update API docs if endpoints change

## Code Style

- Python: Follow PEP 8, use type hints
- Rust: Follow standard Rust conventions, use `cargo fmt` and `cargo clippy`
- TypeScript: Strict mode, use functional components with hooks
